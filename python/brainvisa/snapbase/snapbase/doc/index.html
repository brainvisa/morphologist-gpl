User stories

"As a neuroimager, I want to efficiently control the quality of my raw data, whatever the number of subjects."

"As an image processer in the field of neuroscience, I want an efficient quality control on each step of my processing pipeline on the whole database."

"As a neuroscientist, I want beautiful easy images to present results obtained from big cohorts."

"As a neuroscientist, I want to compare many subjects on a same single screen so that I can detect subtle group patterns and abnormalities."


Features

Generating snapshots of many subjects on a same image

Any type of data is supported, voxel-based or vertex-based, as long as it is viewable in Anatomist software.

Easy selection of subjects, subgroups, or image types, by integration in BrainVisa database system

Fine tuning of the scene and visualization parameters as camera orientation, color maps, slice selection

Code modularity for easy adding of new tools

Python API for batch processing

Configuration

The file snapbase_settings.minf is created in ~/.brainvisa/ when you run SnapBase for the first time.

By simply editing it, various settings can be configured :

default_attributes : data subsets can be filtered according to specific attributes contained in the BrainVisa database (e.g. subject name, protocol name, acquisition name, ...)
  default value : subject, protocol, acquisition

filename_root : each snapshot will have a filename starting with this expression
  default : snapshots_

database_dir : default database to shoot. This is modified through the bottom listbox on the main window.
  default : BrainVisa shared database

output_path : default directory where the snapshots are stored. This value is recalled in the status bar in the main window.
  default : /tmp/
